ids:Message
(core class)		 The following table describes the core ids:Message class with it's properties, which are equal for all messages. 
For communication in the IDS, the specific message types (second table, below) are used and may have new properties as well as additional mandatory fields.
Property Mandatory property Can have multiple values
at the same time		 Description
modelVersion ✓   Information Model version, against which the Message should be interpreted
issued ✓   Date of issuing the message
correlationMessage     Correlated message. Usually needed, if a messages responds to a previous message,
A Connector may, e.g., send a MessageProcessedNotification as a response to an incoming message and therefore needs this property to refer to the incoming message.
issuerConnector     Origin Connector of the message
recipientConnector   ✓ Target Connector
senderAgent ✓   Agent, which initiated the message
recipientAgent   ✓ Agent, for which the message is intended
securityToken ✓   Token representing a claim, that the sender supports a certain security profile
authorizationToken     Authorization token
transferContract     Contract which is (or will be) the legal basis of the data transfer
contentVersion     Version of the content in the payload
